## 背景

当前实现已经在路由、活动栏入口、共享类型、预加载 API、主进程 IPC 处理器、内置文件和项目作用域保存文件中使用工作流命名。部分权威 OpenSpec 规范和项目文档仍在同一领域使用旧名称、旧路由、旧 IPC 域或旧数据模型示例。由于 `openspec/specs/` 和 `AGENTS.md` 会指导后续编码代理，这些引用可能导致代理按错误概念实现。

本变更仅对齐文档和规范，不改变运行时行为、存储布局或 IPC 处理器。

## 目标与非目标

**目标：**

- 让当前 OpenSpec 规范在工作流页面、模板、IPC 域和文档中使用工作流术语。
- 用 `workflow-templates` 替换当前模板能力。
- 从当前规范集合中移除过时的运行时相关规范，而不是按工作流名称重新创建它们。
- 更新 `AGENTS.md` 和文档，使当前指导直接使用工作流且不包含旧术语。

**非目标：**

- 不新增工作流运行列表、运行详情视图、阶段进度可视化或阶段详情能力。
- 不做运行时代码迁移、存储迁移、数据转换或路由别名。
- 不在当前文档中加入解释旧术语的历史说明。

## 决策

1. 将 `workflow-templates` 作为当前模板能力。

   仍符合当前行为的既有模板要求会改写到 `workflow-templates` 下，并使用工作流术语。这样可以让有效的模板行为保持可发现，同时不保留旧能力命名。

2. 从当前规范集合中移除过时的 `pipeline-*` 规范。

   `pipeline-page`、`pipeline-runs`、`pipeline-stage-details` 和 `pipeline-stage-visualization` 描述的是当前工作流功能不包含的运行时概念。实施任务会在对应移除 delta 生效后删除这些规范目录。

3. 通过替换旧示例更新横切规范。

   路由规范应将 `/workflow` 列为项目作用域的工作流页面。IPC 规范应在需要示例时列出 `workflow` 域以及 `workflow:list`、`workflow:save`、`workflow:delete` 通道。通用事件订阅要求会使用与工作流无关的下载示例，避免暗示存在工作流运行时事件面。

4. 将文档和 `AGENTS.md` 作为当前状态指导。

   实施任务会用工作流术语替换旧术语，并移除过时章节。不会增加解释历史或过渡说明。

## 风险与权衡

- 移除过时规范可能使当前 OpenSpec 视图看不到历史原因；历史背景仍保留在 `openspec/changes/archive/`，当前规范继续作为当前行为的权威来源。
- 大范围文本替换可能误改归档背景或生成的依赖文件；实施会将编辑范围限制在 `AGENTS.md`、`docs/` 和当前 `openspec/specs/`，除非任务明确点名其他路径。
- 现有 OpenSpec 全量校验可能包含无关的历史格式问题；本变更的校验聚焦 `openspec validate align-workflow-specs-docs`，并对当前指导内容中的剩余旧术语做定向搜索。
