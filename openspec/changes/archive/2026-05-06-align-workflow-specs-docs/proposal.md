## 为什么

当前可配置自动化的产品概念是“工作流”，但若干权威规范和文档仍使用旧命名。由于 `openspec/specs/` 被视为事实来源，这会误导编码代理。

## 变更内容

- 从 `openspec/specs/` 移除过时的 `pipeline-*` 规范，使其不再被视为当前需求。
- 将当前模板能力从旧命名改为工作流命名。
- 更新仍以旧名称、旧路由、旧通道或旧领域描述工作流相关行为的 OpenSpec 规范。
- 更新项目指导文件和文档，使其直接使用工作流术语，不增加历史弃用说明。
- 本变更不引入工作流运行记录、阶段运行时可视化或阶段运行时详情规范。

## 能力

### 新增能力

- `workflow-templates`：工作流模板列表、模板选择、新建模板入口和来源分组。

### 变更能力

- `app-shell-routing`：将路由示例和受保护路由要求替换为当前工作流路由。
- `project-page-routing`：将项目页面路由要求替换为当前工作流路由。
- `ipc-protocol`：将当前业务域清单替换为包含工作流的清单。
- `ipc-request-response`：将资源示例和文件示例替换为工作流示例。
- `ipc-streaming`：从当前要求中移除与工作流无关的运行时事件示例。
- `integration-tool-registry`：将自动化相关措辞替换为工作流术语。
- `welcome-page`：将产品标语措辞替换为工作流术语。

## 影响

- `openspec/specs/` 下的 OpenSpec 文件。
- `AGENTS.md` 中的项目指导内容。
- `docs/` 下的文档，包括架构、IPC 和数据模型引用。
- 本变更不计划改变运行时代码行为。
