# 功能总览

FylloCode 的功能不是按“聊天工具”组织的，而是按一次 Agent 编码任务的治理路径组织的。

## 产品路径

| 页面 | 解决的问题 |
| --- | --- |
| [任务看板](/features/task) | 集中管理任务入口，从本地任务或研发系统任务发起讨论 |
| [对话与执行](/features/chat) | 在同一个项目上下文中与 Agent 对齐问题、生成方案和推进执行 |
| [Proposal 评审](/features/proposal) | 查看 proposal、design、tasks，并执行 Apply & Archive |
| [Workflow 编排](/features/workflow) | 用 YAML 固化团队自己的执行阶段和 Agent 分工 |
| [ACP Agents](/features/agents) | 安装、识别和管理支持 ACP 的 Coding Agent |
| [研发系统集成](/features/integrations) | 连接云效等研发系统，把任务结果回写到现有工具链 |

## 推荐理解方式

先从一个 Task 开始，然后沿着 Chat、Proposal、Apply & Archive 看完整流程。ACP Agents 和研发系统集成是支撑能力，Workflow 编排则用于把团队已经认可的流程固化下来。
